# PortOSC 开发规则（AI/人工统一）

## 目标
在不破坏现有功能的前提下，逐步提升可维护性、稳定性与可扩展性。

## 强制规则
1. **一次性重构原则**：在当前任务中按整体方案完成，不拆分为多个小步重构。
2. **行为兼容优先**：重构默认不改变 UI 行为与通信协议行为。
3. **先读后改**：修改前必须读取目标文件并确认上下文。
4. **线程安全**：涉及 UI 更新必须走线程安全调用。
5. **异常可见**：禁止静默吞异常（除非明确注释原因并有替代处理）。
6. **避免重复逻辑**：状态切换、IP/端口拼接解析、日志输出等必须尽量复用。
7. **命名清晰**：新增类和方法应体现职责，不引入缩写歧义。
8. **提交规则**：每次修改完成后仅允许本地 `git commit`，禁止 `push`。
9. **版本规则**：当前基线为 `1.0.0`，每轮改造需显式记录版本变更策略。

## 代码组织规则
1. `forms/`：仅放 UI 相关代码，减少业务堆叠。
2. `src/`：协议、服务、工具类。
3. 新增复杂逻辑优先拆到 `src/`，`Form1` 只负责协调。
4. 单文件过大时优先拆分类（partial 或独立 service）。

## 提交流程建议
1. 描述问题与影响范围
2. 给出改动计划
3. 实施并构建验证
4. 本地提交（不推送）
5. 记录结果与后续风险

## 暂定重构顺序
1. `Form1` 内重复 UI 状态逻辑收敛
2. 数据解析与展示职责分离
3. 通信通道统一抽象
4. 补充关键功能测试
